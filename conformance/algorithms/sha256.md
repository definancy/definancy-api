# sha256

> **Vectors:** `vectors/sha256/` *(planned)*
> **Spec:** NIST FIPS 180-4 (SHA-256)

## Algorithm

FIPS 180-4 SHA-256. Used to hash the canonical JWK JSON when computing the
RFC 7638 thumbprint.

## Inputs / outputs

- Input: arbitrary bytes.
- Output: 32-byte digest.

## Java impl

- `Digester.java:13-15` `sha256(data)` delegates to
  `DigestUtils.sha256(data)`.
- Used in `Jwk.java:41` (thumbprint).

## TS impl

- `digester.ts:13-16` `sha256(data)` calls
  `crypto.subtle.digest("SHA-256", data)`.

## Divergence notes

- WebCrypto natively supports `SHA-256`, so the TS implementation works.
- Byte-level output is identical given identical input bytes.
- The only practical risk is upstream: if the canonical JSON fed to SHA-256
  differs between the two languages (see [`jwk_thumbprint.md`](jwk_thumbprint.md)),
  the digests diverge.
